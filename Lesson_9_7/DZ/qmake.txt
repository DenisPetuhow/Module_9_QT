Разобрал способ запуска проекта через .pro файл.Пользователь хочет узнать, как запустить проект через .pro файл (qmake) вместо CMake в Qt Creator.Запуск через .pro файл (qmake)
Создай файл PreGraphic.pro в папке проекта:
pro# Настройки Qt
QT       += core gui concurrent printsupport
greaterThan(QT_MAJOR_VERSION, 4): QT += widgets

# Стандарт C++
CONFIG += c++17

# Имя проекта
TARGET = PreGraphic
TEMPLATE = app

# Исходники
SOURCES += \
    main.cpp \
    mainwindow.cpp \
    graphwindow.cpp

# Заголовки
HEADERS += \
    mainwindow.h \
    graphwindow.h

# Формы UI
FORMS += \
    mainwindow.ui

# ===== QCUSTOMPLOT =====
# Вариант 1: Динамическая библиотека (.dll)
DEFINES += QCUSTOMPLOT_USE_LIBRARY
LIBS += -L$$PWD -lqcustomplot2

# Вариант 2: Статическая (раскомментируй если нужно)
# SOURCES += qcustomplot.cpp
# HEADERS += qcustomplot.h
# # Для MinGW (большие объектные файлы)
# QMAKE_CXXFLAGS += -Wa,-mbig-obj

├── graphwindow.cpp

├── qcustomplot.h          ← Заголовок

Как открыть в Qt Creator

Файл → Открыть файл или проект
Выбери PreGraphic.pro
Выбери комплект сборки (Kit)
Нажми Настроить проект
Собери и запусти (Ctrl+R)

proQT       += core gui concurrent printsupport
